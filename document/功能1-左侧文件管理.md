# 功能 1：左侧文件管理（Sidebar）

## 1. 功能概述

固定 256px 宽度的文件管理面板，提供工作区选择、文件树浏览、文件 CRUD、右键菜单操作。

## 2. 涉及文件

### 渲染进程

| 文件 | 职责 |
|------|------|
| `src/components/sidebar/Sidebar.tsx` | 容器组件：工具栏 + 文件列表 + 菜单动作处理 |
| `src/components/sidebar/FileTree.tsx` | 递归渲染 `FileNode[]` |
| `src/components/sidebar/FileItem.tsx` | 单个文件行：点击打开、右键菜单、内联重命名 |
| `src/components/sidebar/Folder.tsx` | 可折叠文件夹，DRAFTS/ARCHIVES 有特殊标题样式 |
| `src/components/sidebar/CurrentFileInfo.tsx` | 底部面板：文件名 + 字数 + 阅读时间 |
| `src/components/dialogs/RenameDialog.tsx` | 内联重命名输入框 |

### 主进程

| 文件 | 职责 |
|------|------|
| `electron/ipc/fileSystem.ts` | file:read/write/delete/rename/list/create + workspace:open/get/init + chokidar 监听 |
| `electron/ipc/contextMenu.ts` | 原生右键菜单（文件/文件夹/编辑器三类） |
| `electron/ipc/import.ts` | import:files / import:folder / import:drop |

## 3. 实施细节

### 3.1 工作区管理

**首次启动流程**：
1. `App.tsx` 挂载 → 调用 `workspace:get` 读取上次工作区
2. 若为 `null` → 侧边栏显示"尚未选择工作区"空状态 + "选择文件夹"按钮
3. 用户点击 → 触发 `workspace:open` → `dialog.showOpenDialog({ properties: ['openDirectory'] })`
4. 选中后 → `initWorkspace(path)` 创建 DRAFTS / ARCHIVES / .digitalzen 子目录
5. 保存路径到 config → 启动 chokidar 监听 → 返回路径
6. 渲染进程 `setWorkspace(path)` → `file:list(path)` → `setFiles(nodes)`

**工作区目录结构**（自动创建）：
```
workspace/
├── DRAFTS/
├── ARCHIVES/
└── .digitalzen/
    ├── images/
    └── history/
```

### 3.2 文件树渲染

**数据流**：
```
workspace:open/get → file:list(workspace) → FileNode[] → setFiles(zustand)
                                                              ↓
Sidebar → files.find('DRAFTS'/'ARCHIVES') → isSection Folder
       → 其余文件/文件夹 → 普通 Folder / FileItem
```

**FileNode 结构**（`src/types/file.ts`）：
```typescript
interface FileNode {
  name: string;        // 含扩展名
  path: string;        // 绝对路径
  type: 'file' | 'folder';
  children?: FileNode[];
  modifiedAt: string;  // ISO 8601
  size?: number;       // bytes
}
```

**listDir 递归逻辑**（`fileSystem.ts`）：
- 跳过 `.` 开头的隐藏文件/目录
- 仅包含 `.md` 文件 + 子目录
- 按 `modifiedAt` 倒序排列

**DRAFTS/ARCHIVES 特殊处理**：
- `Sidebar.tsx` 从 `files` 中提取 name 为 `DRAFTS`/`ARCHIVES` 的文件夹
- 以 `isSection` 模式渲染（10px 大写字体、letter-spacing 1px、灰色）
- 其余文件/文件夹正常渲染

### 3.3 文件操作

| 操作 | 触发方式 | IPC 调用 | 说明 |
|------|----------|----------|------|
| 打开文件 | 单击 FileItem | `file:read(path)` → `setCurrentFile(path, content)` | 同时 `addRecentFile(path)` |
| 新建文件 | 工具栏"+"按钮 | `file:create(DRAFTS, 'untitled-{timestamp}')` | 创建后立即进入重命名状态 |
| 重命名 | 右键菜单 → "重命名" | 显示 RenameDialog → `file:rename(old, new)` | Enter 确认 / Esc 取消 / blur 取消 |
| 删除 | 右键菜单 → "删除" | `window.confirm()` → `file:delete(path)` | 使用 `shell.trashItem`（移至废纸篓） |
| 移至归档 | 右键菜单 → "移至 Archives" | `file:rename(src, workspace/ARCHIVES/filename)` | |
| 导入文件 | import:files / import:folder | `dialog.showOpenDialog` → 复制到工作区 | 同名文件追加 `_1`, `_2` 后缀 |

### 3.4 右键菜单

**contextMenu:show IPC**：渲染进程传入 `{ type, data }`，主进程构建原生 `Menu` 弹出。

**文件右键**（`isFolder: 'false'`）：
```
打开 → menu:action { type: 'openFile', payload }
重命名 → menu:action { type: 'rename', payload }
──分隔──
移至 Archives → menu:action { type: 'moveToArchives', payload }
──分隔──
在 Finder 中显示 → shell.showItemInFolder (直接执行，不回传)
──分隔──
删除 → menu:action { type: 'delete', payload }
```

**文件夹右键**（`isFolder: 'true'`）：
```
新建文件 → menu:action { type: 'newFile', payload }
新建子文件夹 → menu:action { type: 'newFolder', payload }
──分隔──
重命名 / 在 Finder 中显示 / 删除
```

**菜单动作处理**：`Sidebar.tsx` 的 `useEffect` 监听 `onMenuAction`，根据 `action.type` 分发处理。

### 3.5 文件监听

**chokidar 配置**（`fileSystem.ts`）：
```typescript
chokidar.watch(workspacePath, {
  ignored: [/(^|[/\\])\./, /node_modules/],
  ignoreInitial: true,
  depth: 10,
  awaitWriteFinish: { stabilityThreshold: 300, pollInterval: 100 },
});
```

**事件 → 渲染进程**：
- `.md` 文件的 add/change/unlink 触发 `win.webContents.send('file:changed', { type, path })`
- `App.tsx` 的 `onFileChanged` 回调重新 `file:list` 刷新文件树

### 3.6 样式规范

| 元素 | 规格 |
|------|------|
| 侧边栏宽度 | `256px` 固定 |
| 背景 | `var(--bg-light)` (#F9F9F9) |
| 右边框 | `1px solid var(--border-light)` |
| 顶部留白 | `52px`（macOS Traffic Lights 区域，`-webkit-app-region: drag`）|
| 文件项高度 | `36px` |
| 文件项字体 | `var(--font-code)` 12px |
| 激活态 | 黑底白字，`var(--radius-sm)` 圆角 |
| Hover 态 | `var(--bg-hover)` |
| 分类标题 | `var(--font-ui)` 10px，大写，letter-spacing 1px，灰色 |

### 3.7 状态依赖

从 Zustand store 消费：
```
useFiles()          → FileNode[] 文件树
useWorkspace()      → string | null 当前工作区路径
useCurrentFile()    → string | null 当前打开文件
useCurrentContent() → string 当前文件内容（用于字数统计）
useAppStore(s => s.setWorkspace / setFiles / setCurrentFile / addRecentFile)
```

监听主进程事件：
```
window.electronAPI.onFileChanged → 刷新文件列表
window.electronAPI.onMenuAction  → 处理右键菜单动作
```
