## v0.2.1 — 用户反馈紧急修复

> 来源：实际使用反馈（2026-04-17）

### F1. 添加主题切换按钮 ⭐

**问题**：当前主题切换只能通过设置面板（Cmd+,），没有直观的切换按钮。
**修复**：在侧边栏底部或工具栏添加主题切换图标按钮（太阳/月亮），点击轮转 light → dark → auto。
**涉及文件**：`Sidebar.tsx`

### F2. 图片无法预览 ⭐

**问题**：编辑模式插入的本地图片在阅读模式无法显示。
**原因**：Electron 的 CSP 默认阻止 `file://` 协议图片加载。
**修复**：
- `electron/main.ts` 创建窗口时设置 `webSecurity: false` 或自定义 CSP 允许 `file:` 和 `data:` 协议
- 确认 `rehypeFixImagePaths` 正确拼接 `file://` 路径
**涉及文件**：`electron/main.ts`, `src/lib/markdown.ts`

### F3. 无法创建文件夹 ⭐

**问题**：侧边栏只有"新建文件"按钮，没有"新建文件夹"功能。
**修复**：
- 工具栏添加"新建文件夹"按钮
- 主进程添加 `file:createDir` IPC handler（`fs.mkdir`）
- 创建后进入重命名状态
**涉及文件**：`Sidebar.tsx`, `electron/ipc/fileSystem.ts`, `electron/preload.ts`, `src/types/ipc.ts`

### F4. 移除编辑器底部信息栏

**问题**：右下角的字数和阅读时间不需要。
**修复**：EditorArea.tsx 中移除 `<FileInfoBar />` 组件引用。
**涉及文件**：`src/components/editor/EditorArea.tsx`

### F5. 默认打开编辑模式

**问题**：切换到阅读模式后持久化了 `viewMode: 'read'`，下次启动仍为阅读模式。
**修复**：从 persist 的 `partialize` 中移除 `viewMode`，使每次启动都回到 `'edit'`。
**涉及文件**：`src/stores/appStore.ts`

### F6. 砍掉版本历史功能 ⭐

**原因**：功能过重，当前阶段不需要。
**修复**：
- 删除 `src/components/history/HistoryPanel.tsx` + `.module.css`
- 删除 `electron/ipc/history.ts`
- `electron/main.ts` 移除 `registerHistoryHandlers`
- `electron/preload.ts` 移除 `history` API
- `src/types/ipc.ts` 移除 `HistoryAPI`
- `App.tsx` 移除 Cmd+Shift+H 快捷键 + `<HistoryPanel />` 渲染
- `EditorArea.tsx` 移除 `history:save` 调用
- `appStore.ts` 可保留 `historyOpen` 字段（不影响，或一并清除）

### F7. 设置面板内添加"使用说明" ⭐

**问题**：用户不知道快捷键和功能用法，需要一份内置使用说明。
**修复**：
- 设置面板新增第 5 个分区 **"使用说明"**
- 内容为内嵌的 Markdown 渲染（复用 `renderMarkdown`），包含：
  - 基本操作（选择工作区、创建/打开/编辑/保存文件）
  - 快捷键表格（Cmd+E/P/S/B/I/K/,）
  - Markdown 语法速查（标题/粗体/斜体/链接/图片/代码/表格/列表）
  - 主题切换说明
  - 导出说明
- 或者直接在设置面板中渲染一个只读的 `.md` 文件
**涉及文件**：`src/components/settings/sections/index.tsx`, `SettingsPanel.tsx`

### F8. 阅读模式暗色链接不完整

**问题**：阅读模式链接颜色 `#0066cc` 在暗色下不可读。
**修复**：`reader.css` 补充 `[data-theme="dark"] .dz-reader a { color: #6cb6ff; }`
**涉及文件**：`src/styles/reader.css`

### F9. ModeSwitch 无文件时应禁用

**问题**：未选择文件时 READ/EDIT 按钮可点击。
**修复**：`currentFile === null` 时 disable 或隐藏按钮。
**涉及文件**：`src/components/editor/ModeSwitch.tsx`

### F10. 清理死代码

- `EditorArea.module.css` / `MarkdownRenderer.module.css` 中的 `.dragArea` 类
- `SettingsPanel.module.css` 中 toggle 的 `white` → `var(--bg-white)`

### F11. 所有图标按钮添加 Tooltip

**问题**：侧边栏工具栏、主题切换、导出按钮等图标按钮没有文字说明，用户不知道功能。
**修复**：所有 `iconBtn` 类按钮确保有 `title` 属性（浏览器原生 tooltip），内容为 "功能名 (快捷键)"。
**涉及文件**：`Sidebar.tsx`, `ModeSwitch.tsx`, `ExportBar.tsx`, `SettingsPanel.tsx`

### F12. 搜索功能改为编辑区全局搜索 ⭐

**问题**：当前 Cmd+P 是文件名模糊搜索弹窗（fuse.js），实际需求是在右侧 MD 编辑区内做全局搜索（查找/替换）。
**修复**：
- 砍掉 `SearchModal.tsx`（文件名搜索弹窗）及其 fuse.js 依赖
- 改为 CodeMirror 内置的 `@codemirror/search` 扩展（Cmd+F 查找、Cmd+H 替换）
- 支持：高亮匹配、逐个跳转、全部替换、正则可选
- `App.tsx` 移除 Cmd+P / openSearch 快捷键
- `appStore.ts` 的 `searchOpen` 状态可保留或清除
**涉及文件**：
- 删除：`src/components/search/SearchModal.tsx`, `SearchModal.module.css`
- 修改：`src/components/editor/EditorExtensions.ts`（添加 `@codemirror/search` 扩展）
- 修改：`src/App.tsx`（移除 Cmd+P）
- `package.json`：可移除 `fuse.js` 依赖
